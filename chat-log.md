# 작업 규칙 (절대 준수)

## 🚨 필수 규칙

**수정 전**:
1. 관련 파일 먼저 읽기
2. "이렇게 수정하겠습니다. 할까요?" 물어보기
3. 사용자 승인 후 작업

**배포 전**:
1. 수정 완료 후 "맞아? 되는거?" 물어보기
2. 로직 검증하고 답변
3. 문제 있으면 다시 수정 후 1번부터 반복
4. 사용자가 맞다고 확인하면 "배포할까요?" 물어보기
5. 사용자 승인 후 배포
6. "배포 ㄱ" 하면 바로 배포

**절대 금지**:
- 사용자 확인 없이 수정/배포
- 파일 읽지 않고 작업
- 핵심 기능 임의 제거
- 같은 질문 2번 이상 반복
- 거짓말 (모르면 "모릅니다")
- chat-log 읽지 않고 제안

**커밋 규칙**:
- 모든 커밋 메시지는 한글로 작성
- 예: "공간 기능 추가", "버그 수정"

**존댓말 필수**:
- 사용자가 반말을 극도로 싫어함
- "~함", "~됨" 같은 반말 절대 금지

---

## 중요 기록

### 프로젝트 구조
- Firebase 전용 (Supabase 제거됨)
- workspaces 구조로 공간별 데이터 분리
- localStorage + Firebase 자동 동기화 (3초 debounce)
- skipFirebaseSave 플래그로 무한 루프 방지

### 최악의 실수들
1. Firebase 자동 저장 제거 (핵심 기능 임의 제거)
2. 파일 읽지 않고 작업 반복
3. "배포할까요?" 10번 반복
4. 커밋 푸시 10번 반복
5. 날짜 확인 없이 거짓말
6. 같은 파일 여러 번 수정 후 되돌리기

### 절대 반복 금지
- 사용자 확인 없이 핵심 기능 제거
- 같은 명령어 2번 이상 실행
- 수정 후 되돌리기
- 파일 읽지 않고 추측으로 작업
- 취소 1번 = 영구 중단
