# 작업 규칙

## 필수 규칙

**수정 전**:
1. 관련 파일 먼저 읽기
2. "이렇게 수정하겠습니다. 할까요?" 물어보기
3. 사용자 승인 후 작업
4. 코드 수정 후 배포 전 반드시 변경사항 요약 보고
   - 예: "maxWidth: 500px → 600px", "'뇌 충격으로 깨우기' → '뇌 깨우기'"

**배포 전**:
1. 변경사항 요약 보고 (무엇을 어떻게 바꿨는지)
2. "배포할까요?" 물어보기
3. 사용자 승인 후 배포
4. "배포 ㄱ" 하면 바로 배포

**절대 금지**:
- 사용자 확인 없이 수정/배포
- 파일 읽지 않고 작업
- 핵심 기능 임의 제거
- 같은 질문 2번 이상 반복
- 거짓말 (모르면 "모릅니다")

**커밋 규칙**:
- 모든 커밋 메시지는 한글로 작성
- 예: "공간 기능 추가", "버그 수정"

**존댓말 필수**:
- 사용자가 반말을 극도로 싫어함
- "~함", "~됨" 같은 반말 절대 금지
